# QA Checklist

## 실행 전 확인

- `npm.cmd install`이 완료되어 있다
- `node -v`가 정상 출력된다
- `4000`, `5173` 포트가 비어 있다

## 기능 체크

- `npm.cmd run dev`로 프론트와 백엔드가 함께 실행된다
- 브라우저에서 `http://127.0.0.1:5173` 접속이 가능하다
- 초기 목록이 정상 표시된다
- 새 Todo 생성이 가능하다
- Todo 상세 보기로 이동 가능하다
- Todo 수정이 가능하다
- 완료 상태 토글이 가능하다
- Todo 삭제가 가능하다

## 예외 체크

- 제목 없이 생성 시 프론트에서 검증 메시지가 보인다
- 잘못된 요청 시 백엔드가 400을 반환한다
- 존재하지 않는 Todo 접근 시 404가 반환된다
- 서버를 내린 상태에서 새로고침하면 프론트 에러 배너가 보인다

## 디버깅 체크

- 브라우저 콘솔에 `api:request`, `api:response`, `ui:submit` 로그가 보인다
- 백엔드 로그에 `requestId`와 요청 시작/종료 로그가 남는다
- 오류 응답에 `requestId`가 포함된다
- 화면 하단 `Debug Snapshot`이 현재 상태를 반영한다

## 테스트 체크

- `npm.cmd test`가 통과한다
- `npm.cmd run test:e2e`가 통과한다
