프로젝트 목표

- 지도 기반 약속 스케줄러를 만드는 것이 목표입니다.
- 백엔드는 Express + TypeScript + Prisma + SQLite를 사용합니다.
- 프론트엔드는 React + TypeScript + Vite를 사용합니다.
- MVP는 appointments CRUD부터 시작합니다.
- places는 출발지/도착지로 재사용 가능한 장소 데이터입니다.
- 이후에는 routes 기능과 네이버 지도 API 연동을 붙일 예정입니다.
- 전체 코드베이스는 초보자도 이해 가능한 수준으로 유지하고 싶습니다.

현재 전체 상태

- 백엔드와 프론트엔드가 모두 존재합니다.
- 배치 파일로 앱을 한 번에 실행할 수 있습니다.
- 모바일 우선 대시보드 UI가 있습니다.
- appointments CRUD가 구현되어 있습니다.
- places 생성 / 조회 / 수정 / 삭제가 구현되어 있습니다.
- Prisma Studio로 DB 관리가 가능합니다.
- saved routes와 네이버 지도 API는 아직 구현되지 않았습니다.

현재 Prisma schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

enum TransportMode {
  WALK
  CAR
  TRANSIT
  BICYCLE
}

enum AppointmentStatus {
  SCHEDULED
  COMPLETED
  CANCELED
}

model User {
  id           String        @id @default(cuid())
  email        String        @unique
  passwordHash String
  name         String
  createdAt    DateTime      @default(now())
  updatedAt    DateTime      @updatedAt

  appointments Appointment[]
  tags         Tag[]
}

model Place {
  id            String        @id @default(cuid())
  name          String
  roadAddress   String?
  jibunAddress  String?
  lat           Float
  lng           Float
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  originAppointments      Appointment[] @relation("OriginPlace")
  destinationAppointments Appointment[] @relation("DestinationPlace")

  originRoutes      SavedRoute[] @relation("RouteOriginPlace")
  destinationRoutes SavedRoute[] @relation("RouteDestinationPlace")

  @@index([lat, lng])
  @@index([name])
}

model Appointment {
  id                 String             @id @default(cuid())
  userId             String
  title              String
  memo               String?
  startAt            DateTime
  endAt              DateTime?
  transportMode      TransportMode
  status             AppointmentStatus  @default(SCHEDULED)
  originPlaceId      String?
  destinationPlaceId String
  createdAt          DateTime           @default(now())
  updatedAt          DateTime           @updatedAt

  user             User             @relation(fields: [userId], references: [id], onDelete: Cascade)
  originPlace      Place?           @relation("OriginPlace", fields: [originPlaceId], references: [id], onDelete: SetNull)
  destinationPlace Place            @relation("DestinationPlace", fields: [destinationPlaceId], references: [id], onDelete: Restrict)

  routes          SavedRoute[]
  appointmentTags AppointmentTag[]

  @@index([userId, startAt])
  @@index([destinationPlaceId])
  @@index([originPlaceId])
  @@index([status])
}

model SavedRoute {
  id                 String   @id @default(cuid())
  appointmentId      String
  originPlaceId      String
  destinationPlaceId String
  summary            String?
  distanceMeters     Int
  durationSeconds    Int
  routePolylineJson  String?
  selectedOption     Boolean  @default(false)
  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt

  appointment      Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  originPlace      Place       @relation("RouteOriginPlace", fields: [originPlaceId], references: [id], onDelete: Restrict)
  destinationPlace Place       @relation("RouteDestinationPlace", fields: [destinationPlaceId], references: [id], onDelete: Restrict)

  @@index([appointmentId])
  @@index([originPlaceId])
  @@index([destinationPlaceId])
  @@index([selectedOption])
}

model Tag {
  id        String   @id @default(cuid())
  userId    String
  name      String
  color     String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user            User             @relation(fields: [userId], references: [id], onDelete: Cascade)
  appointmentTags AppointmentTag[]

  @@unique([userId, name])
}

model AppointmentTag {
  appointmentId String
  tagId         String

  appointment Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  tag         Tag         @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([appointmentId, tagId])
  @@index([tagId])
}
```

schema 관련 참고사항

- schema는 처음 초안과 최대한 가깝게 유지하고 있습니다.
- `SavedRoute.transportMode`는 아직 추가하지 않았습니다.
- 하지만 나중에 필요해질 가능성이 높다고 보고 있습니다.
- 이걸 지금 추가하는 게 좋은지, 나중에 추가하는 게 좋은지도 의견이 필요합니다.

현재 백엔드 구조

```text
server/
  prisma/
    schema.prisma
    seed.ts
  src/
    app.ts
    index.ts
    config/
      env.ts
    lib/
      prisma.ts
    middlewares/
      error-handler.ts
      request-context.ts
    modules/
      appointments/
        appointments.controller.ts
        appointments.routes.ts
        appointments.schema.ts
        appointments.service.ts
      places/
        places.controller.ts
        places.routes.ts
        places.schema.ts
        places.service.ts
    utils/
      http-error.ts
      logger.ts
```

백엔드 설계 의도

- controller -> service 구조로 단순하게 유지하고 있습니다.
- Prisma를 쓰기 때문에 repository 계층은 일부러 두지 않았습니다.
- 초보자 입장에서 파일 수가 너무 많아지는 것을 피하고 싶었습니다.
- 입력 검증은 Zod를 사용합니다.
- 에러는 HttpError와 공통 error middleware로 통일합니다.

현재 백엔드 API

- GET /api/appointments?userId=...
- GET /api/appointments/:id
- POST /api/appointments
- PATCH /api/appointments/:id
- DELETE /api/appointments/:id
- GET /api/places
- POST /api/places
- PATCH /api/places/:id
- DELETE /api/places/:id

백엔드 동작 관련 참고사항

- 아직 인증이 없어서 `userId`를 직접 전달하고 있습니다.
- appointment는 `destinationPlaceId`가 필수입니다.
- place 삭제는, 현재 appointment의 도착지로 쓰이는 경우나 saved route에서 참조 중인 경우 막고 있습니다.
- 이 역시 MVP 기준에서 단순하게 유지한 것입니다.

프론트엔드 설계 의도

- 파일 수를 너무 많이 늘리지 않고 한 페이지 중심으로 유지하고 있습니다.
- React Query로 조회와 캐시 무효화를 처리합니다.
- 스타일은 아직 CSS 한 파일에 모아두고 있습니다.
- UI는 모바일 우선으로 조정했습니다.
- 현재 화면에는 다음이 있습니다.
  - appointment 입력/수정 폼
  - appointment 목록
  - place 관리 패널
- 모바일에서는 액션 버튼이 하단에서 쉽게 눌리도록 배치했습니다.
- place는 카드 선택 -> 편집 -> 삭제 흐름으로 동작합니다.

현재 배치 파일

```text
run-app.bat
verify-all.bat
```

배치 파일 설명

- `run-app.bat`
  - `.env`가 없으면 자동 생성
  - 의존성이 없으면 설치
  - Prisma client가 필요하면 생성
  - DB가 없으면 push + seed
  - dev 서버 실행
  - 브라우저 자동 열기

- `verify-all.bat`
  - server lint/build
  - client lint/test/build
  - 전체 검증을 한 번에 수행
