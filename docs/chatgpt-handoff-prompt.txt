Project goal

- Build a map-based appointment scheduler.
- Backend uses Express + TypeScript + Prisma + SQLite.
- Frontend uses React + TypeScript + Vite.
- MVP starts from appointments CRUD.
- Places are reusable origin/destination records.
- Later, route calculation and Naver Map API integration should be added.
- The codebase should stay beginner-friendly and not too fragmented.

Current high-level state

- Backend and frontend both exist.
- App can be launched with a batch file.
- There is a mobile-first dashboard UI.

- Places create / list / update / delete is implemented.
- Prisma Studio can be used for DB management.
- Saved routes and Naver Map API are not implemented yet.

Current Prisma schema

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

enum TransportMode {
  WALK
  CAR
  TRANSIT
  BICYCLE
}

enum AppointmentStatus {
  SCHEDULED
  COMPLETED
  CANCELED
}

model User {
  id           String        @id @default(cuid())
  email        String        @unique
  passwordHash String
  name         String
  createdAt    DateTime      @default(now())
  updatedAt    DateTime      @updatedAt

  appointments Appointment[]
  tags         Tag[]
}

model Place {
  id            String        @id @default(cuid())
  name          String
  roadAddress   String?
  jibunAddress  String?
  lat           Float
  lng           Float
  createdAt     DateTime      @default(now())
  updatedAt     DateTime      @updatedAt

  originAppointments      Appointment[] @relation("OriginPlace")
  destinationAppointments Appointment[] @relation("DestinationPlace")

  originRoutes      SavedRoute[] @relation("RouteOriginPlace")
  destinationRoutes SavedRoute[] @relation("RouteDestinationPlace")

  @@index([lat, lng])
  @@index([name])
}

model Appointment {
  id                 String             @id @default(cuid())
  userId             String
  title              String
  memo               String?
  startAt            DateTime
  endAt              DateTime?
  transportMode      TransportMode
  status             AppointmentStatus  @default(SCHEDULED)
  originPlaceId      String?
  destinationPlaceId String
  createdAt          DateTime           @default(now())
  updatedAt          DateTime           @updatedAt

  user             User             @relation(fields: [userId], references: [id], onDelete: Cascade)
  originPlace      Place?           @relation("OriginPlace", fields: [originPlaceId], references: [id], onDelete: SetNull)
  destinationPlace Place            @relation("DestinationPlace", fields: [destinationPlaceId], references: [id], onDelete: Restrict)

  routes          SavedRoute[]
  appointmentTags AppointmentTag[]

  @@index([userId, startAt])
  @@index([destinationPlaceId])
  @@index([originPlaceId])
  @@index([status])
}

model SavedRoute {
  id                 String   @id @default(cuid())
  appointmentId      String
  originPlaceId      String
  destinationPlaceId String
  summary            String?
  distanceMeters     Int
  durationSeconds    Int
  routePolylineJson  String?
  selectedOption     Boolean  @default(false)
  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt

  appointment      Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  originPlace      Place       @relation("RouteOriginPlace", fields: [originPlaceId], references: [id], onDelete: Restrict)
  destinationPlace Place       @relation("RouteDestinationPlace", fields: [destinationPlaceId], references: [id], onDelete: Restrict)

  @@index([appointmentId])
  @@index([originPlaceId])
  @@index([destinationPlaceId])
  @@index([selectedOption])
}

model Tag {
  id        String   @id @default(cuid())
  userId    String
  name      String
  color     String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  user            User             @relation(fields: [userId], references: [id], onDelete: Cascade)
  appointmentTags AppointmentTag[]

  @@unique([userId, name])
}

model AppointmentTag {
  appointmentId String
  tagId         String

  appointment Appointment @relation(fields: [appointmentId], references: [id], onDelete: Cascade)
  tag         Tag         @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([appointmentId, tagId])
  @@index([tagId])
}
```

Important note about the schema

- The schema is intentionally close to the original draft.
- `SavedRoute.transportMode` has not been added yet, even though that may become necessary later.
- I want feedback on whether to change that now or later.

Backend design notes

- It uses a simple controller -> service pattern.
- There is no repository layer because Prisma is used directly in services to keep things simple.
- Validation uses Zod.
- Errors are normalized with a shared HttpError + error middleware.
- Current APIs:
  - GET /api/appointments?userId=...
  - GET /api/appointments/:id
  - POST /api/appointments
  - PATCH /api/appointments/:id
  - DELETE /api/appointments/:id
  - GET /api/places
  - POST /api/places
  - PATCH /api/places/:id
  - DELETE /api/places/:id

Backend behavior notes

- `userId` is still passed directly because authentication is not implemented yet.
- `destinationPlaceId` is required for appointments.
- Place deletion is blocked if the place is used as a destination in appointments or is referenced by saved routes.
- This is intentionally still MVP-oriented.

Batch file notes

- `run-app.bat`
  - prepares `.env` if missing
  - installs dependencies if needed
  - generates Prisma client if needed
  - pushes DB and seeds when DB is missing
  - starts the dev servers
  - opens the browser

- `verify-all.bat`
  - runs server lint/build
  - runs client lint/test/build
